# TASK-012: A/B eval vs FIFO baseline + demo script + metrics

A status change is written in BOTH places at once: the `status:` field above, and this task's row
in `docs/tasks/master-plan.md`. They must never disagree. Read the board row back after writing it.

## Goal

Retargeted 2026-07-18 (pitch pivot, see Decisions and blockers): the commercial thesis is no longer
"agent pipeline beats FIFO scheduling." It is "the patient goes from zero visibility into station
queues (today's reality: blind wandering, backtracking between blood draw / X-ray / consult) to an
informed, AI-recommended route (FR-23: aggregated per-station wait + shortest-queue-first routing
with parallelization)." Run that comparison in the SimPy world, report the metrics, produce the
demo script.

## Inputs and context

- Related files and modules: `src/vaic/simulator/` (exclusive owner). Builds on the world + metrics
  harness (TASK-006).
- Consumes FR-23 (TASK-026/027/028): aggregated `estimate_wait()` per station (FR-07) and the
  AI-recommended, shortest-queue-first, parallelized route generated by the Care Plan Agent
  (FR-04/FR-08). Does NOT depend on TASK-010 (Coordinator/Disruption) - FR-23 routing lives entirely
  in the Care Plan Agent, already built/in progress on TASK-027.
- No longer depends on TASK-010 for the headline comparison (dependency dropped; master-plan row to
  be updated to reflect TASK-006, TASK-027 as the real deps). TASK-010/TASK-029 remain relevant only
  if a secondary multi-patient disruption-handling metric is added later.
- Feeds the coordinator console metrics panel (TASK-029, if still in scope) and the `judge`
  evaluation.

## To do

- [ ] A/B harness, two arms on the same seeded world:
      - Baseline ("today"): patient follows a fixed/default station order with no visibility into
        per-station queues and no re-routing - models blind wandering, including backtrack/detour
        time when a station turns out to be congested.
      - Treatment ("VAIC"): patient receives FR-23's aggregated per-station wait + AI-recommended
        route, routed to the shortest queue at each step, parallelized where stations are
        independent.
- [ ] Headline metric: patient time-in-system (total wait + service + backtrack time across all of
      a patient's stations for one visit). Secondary metric: station visits/backtracks avoided.
      Station utilisation and ETA MAE are retained as supporting metrics, not the headline.
- [ ] Frozen, versioned eval case set including the known-hard cases and a safety slice
      (injection, refusal, past-wrong-answer inputs) per `.claude/rules/ai-governance.md`.
- [ ] Demo script: the click-through that shows the win.

## Acceptance criteria

- [ ] A/B run reproducible from a committed seed; results recorded in this task file.
- [ ] VAIC arm beats the no-visibility baseline on patient time-in-system, or the gap is explained.
- [ ] A regression on the safety slice blocks the result, whatever the average did.

## Decisions and blockers

- 2026-07-18: Pitch pivot from "agent pipeline beats FIFO" to "patient goes from no station-queue
  visibility to an AI-guided route" (business-viability driver: public hospitals cannot staff a
  dedicated patient coordinator; the product substitutes for that role at zero marginal cost).
  Consequence: this task's baseline, headline metric, and deps changed (see Goal, Inputs, To do).
  Flagged to `spec-guardian` for a requirement-drift check against FR-23/US-22 and the FR-09/FR-10
  framing elsewhere in the specs, since the pitch no longer centers on FR-09/FR-10 at all.
